Выводы

Автоматика подтверждает 88% результатов операторов — из 282 номеров, найденных 4 операторами, 247 совпали с данными API

Лучший recall у Юлии (99%) и Дианы (95%). Елена показала 71% — 19 расхождений, стоит разобрать вручную

1138 сайтов (67%) — звонки от компаний были, но операторы их не обработали. Это главный потенциал автоматизации

На 272 сайта (16%) никто не перезвонил за 4 дня — заявки не сработали

Всего 29 расхождений на 1692 записи — нужно разобрать вручную для понимания причин (ошибка оператора или автоматики)

На один номер приходит до 21 записи с речью (среднее 3.2). Одна SIM-карта используется на нескольких сайтах — звонки от разных компаний смешиваются. Нужен анализ транскрипции для привязки callerid к конкретному сайту

Следующий шаг — подключить LLM для определения callerid→сайт при нескольких звонках с разных компаний на один номер

Методология

Матчинг: сравниваем callerid из API с номером, который записал оператор. Если callerid содержит номер оператора или наоборот — «Совпал»

Записей на листе: до 10 на номер (отсортированы по уверенности транскрипции). Анализируем коротких записей, включая «але», «алло» — среди них часто нужный обратный звонок

Общие SIM-карты: один номер заявки может использоваться на 2-3 сайтах разными операторами. API возвращает все звонки на этот номер — от всех компаний сразу

Потенциал автоматизации

Сейчас 4 оператора находят номера компаний для 17% сайтов (282 из 1692).

API возвращает записи звонков для 84% сайтов (1420 из 1692).

Автоматизация может увеличить сбор ОН в 5 раз

с ~70 до ~355 номеров в день на одного оператора, при подключении LLM-анализа транскрипций
